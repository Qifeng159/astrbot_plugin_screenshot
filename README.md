# 截图插件

对指定显示器进行截图并将图片发送到当前会话，支持多显示器环境。

## 功能说明

- 多种触发指令：`/截图` `/截屏` `/screen` `/screenshot` `/capture` `/printscreen`
- 支持多显示器环境，可自由选择要截图的显示器
- 截图以 PNG 格式发送到当前会话
- 普通用户均可使用截图指令
- 自动清理过期截图（可配置天数）
- 手动清理指令：`/清理截图`（仅管理员）

## 安装与依赖

### 安装

将插件文件夹放入 AstrBot 的 `data/plugins` 目录下，重启 AstrBot 即可。

### 依赖

```bash
pip install mss
```

AstrBot 首次加载插件时会自动安装依赖；若失败请手动执行上述命令。

## 使用指令

### 截图

| 指令 | 权限 | 说明 |
|------|------|------|
| `/截图` | 全体成员 | 对当前配置的显示器截图（默认主显示器） |
| `/截屏` | 全体成员 | 中文别名 |
| `/screen` | 全体成员 | 英文别名 |
| `/screenshot` | 全体成员 | 英文别名 |
| `/capture` | 全体成员 | 英文别名 |
| `/printscreen` | 全体成员 | 英文别名 |

所有指令均可省略前面的 `/`。

### 清理

| 指令 | 权限 | 说明 |
|------|------|------|
| `/清理截图` | 管理员 | 立即删除 temp 目录下所有历史截图 |
| `/cleanscreenshot` | 管理员 | 英文别名 |

## 配置项说明

所有配置项均可在 AstrBot 管理面板的「插件配置」页面中修改。

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `screenshot_command` | string | `"截图"` | 触发关键词（保留项，建议使用内置别名） |
| `monitor_index` | int | `0` | 截图目标显示器编号，0=主显示器 |
| `auto_clean_days` | int | `7` | 自动清理天数，每次截图前清理 N 天前的旧文件。设为 0 关闭自动清理 |

## 多显示器编号说明

| 配置值 | 对应显示器 |
|--------|-----------|
| `0` | 主显示器（默认） |
| `1` | 第二个显示器 |
| `2` | 第三个显示器 |

超出范围自动回退到主显示器。

## 清理机制

- **自动清理**：根据 `auto_clean_days` 配置，每次截图前自动删除超过 N 天的旧截图
- **手动清理**：管理员发送 `/清理截图` 立即清空所有历史截图文件
- 截图存储在插件目录 `temp/` 子文件夹下

## 注意事项

1. 本插件必须在有 GUI 的 Windows 系统上运行
2. 建议以当前登录用户身份运行 AstrBot
3. RDP 断开或锁屏时截图可能返回黑屏
4. 截图直接发送到当前会话，请注意隐私
5. 建议 `auto_clean_days` 设置为 7~30 天，避免 temp 目录膨胀
